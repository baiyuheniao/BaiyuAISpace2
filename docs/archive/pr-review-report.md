|----------|------|------|-------------|
| **Blocking** | mcp.rs:L240-L252 | 参数验证逻辑存在缺陷：未匹配到白名单时不验证危险模式 | 修改逻辑为：先检查危险模式，再检查白名单 |

问题代码分析：
```rust
// 当前逻辑：如果不匹配白名单，才检查危险模式
// 这意味着未在白名单中的"安全"参数会被直接放行
if !is_allowed && !arg.is_empty() {
    // 只检查危险模式
}
```

**建议修复**：
```rust
fn validate_args(&self, args: &[String]) -> Result<(), MCPError> {
    // 先检查危险模式
    let dangerous_chars = ["&&", "||", "|", ">", ">>", "<", "`", "$(", ";", "\n", "\r", "\0"];
    for arg in args {
        for pattern in dangerous_chars {
            if arg.contains(pattern) {
                return Err(MCPError::LaunchError(format!(
                    "Argument '{}' contains dangerous pattern", arg
                )));
            }
        }
    }

    // 然后检查白名单
    for arg in args {
        let is_allowed = self.allowed_args.iter().any(|allowed| {
            arg == *allowed ||
            arg.starts_with("--") ||
            arg.starts_with("-") && arg.len() == 2 ||
            arg.starts_with("npm_package_") ||
            arg.starts_with("npm_config_")
        });

        if !is_allowed && !arg.is_empty() {
            return Err(MCPError::LaunchError(format!(
                "Argument '{}' is not in whitelist", arg
            )));
        }
    }
    Ok(())
}
```

| Major | mcp.rs:L252 | `dangerous_chars` 定义了两次（函数内和 validate_args 内） | 提取为模块级常量避免重复定义 |
