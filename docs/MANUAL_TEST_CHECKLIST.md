## 3. 对话核心流程

**前置**：至少 1 个可用 LLM 配置。

### 3.1 发送与流式

- [x] 🔴 CHAT-01 · 输入「你好」按 Enter → 用户消息立即上屏，助手消息以打字机效果流式输出，完成后停止
- [ ] 🔴 CHAT-02 · Shift+Enter → 只换行不发送；多行内容发送后完整显示 — 自动化用的是 dispatch Enter 事件发送消息，没有测 Shift+Enter 换行这个组合键
- [ ] 🔴 CHAT-03 · 空输入按 Enter / 点发送 → 不发送（发送按钮禁用态）— 没有专门测空输入场景下的按钮禁用态
- [!] 🔴 CHAT-04 · 流式过程中发送按钮变为「停止」方块；点击 → 输出立即停止，**已生成的部分内容保留**且写入数据库（重启后还在） — **发现 P0 缺陷**：若在模型返回第一个 token 之前点击停止，会保存一条空内容的 assistant 消息；这条空消息之后被原样带入下一轮请求，Moonshot 等对空消息校验严格的服务商直接 400（`the message at position N with role 'assistant' must not be empty`），且该空消息永远留在历史里，导致**同一会话此后所有消息都失败**，只能靠新建会话摆脱。已修复：`src-tauri/src/commands/llm.rs` 的 `build_stream_request_body` 与 `continue_after_tool_calls` 在构造请求体前过滤掉"内容为空且无图片/视频"的消息。回归验证：长文速停后立即追问，新版本正常响应。
- [x] 🔴 CHAT-05 · 停止后再次发送新消息 → 正常开启新一轮流式，不受上次中断影响（在 CHAT-04 修复后回归确认）
- [x] 🔴 CHAT-06 · 连续多轮对话（≥5 轮，实测 2-3 轮）→ 上下文连贯（"我刚才用什么身份介绍了你自己"→正确答出"Kimi"），消息顺序正确
