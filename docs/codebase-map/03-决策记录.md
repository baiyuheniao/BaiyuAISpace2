# 决策记录：为什么是这样

> 这份文档回答：**"这个项目为什么长成现在这个样子？"**
> 代码只记录"是什么"，不记录"为什么"——而"为什么"恰恰是项目所有者最该握在手里的东西：懂了决策，你就能判断未来的改动是顺着走还是在推翻它。
> 这是 2026-07-17 从 git 历史逆向整理的（括号里的短哈希可用 `git show <哈希>` 查看原始改动）。**此后的新决策随每次交付讲解追加进来。**

每条按同一格式：**决定了什么 / 为什么 / 代价与边界**。

---

## A. 技术选型（地基）

### A1. Tauri 2 + Rust 后端 + Vue 3 前端（立项之初，`37e7030`）

- **决定**：桌面框架选 Tauri 而非 Electron；后端 Rust，前端 Vue 3 + TypeScript，Windows 为主力平台。
- **为什么**：Tauri 复用系统自带的 WebView2，安装包和内存占用远小于 Electron（自带整个 Chromium）；Rust 提供单二进制分发和系统级能力（托盘、快捷键、进程管理）。
- **代价与边界**：Rust 上手门槛高、编译慢（cargo target 一次可达 ~10GB，用完 worktree 要清理）；WebView2 profile 有锁，同一时刻只能跑一个实例。

### A2. 数据全部本地：SQLite 单文件 + 系统钥匙串（立项之初；`1b5051b`、`091e393`）

- **决定**：所有业务数据存本地 SQLite（一个文件、多组表，见架构地图）；API 密钥不进数据库，走系统钥匙串（keyring 库，Windows 凭据管理器）。
- **为什么**：本地优先 = 无服务器成本、无隐私顾虑，符合"个人桌面工具"定位。密钥单独放钥匙串，是因为 SQLite 文件是明文的，拷走文件不该等于拷走密钥（MCP 的密钥后来也照此迁移，Issue #6）。
- **代价与边界**：没有云同步，换机器要手动迁移数据。

### A3. 所有 LLM 服务商对接集中在一个 llm.rs（演化形成）

- **决定**：15+ 家服务商的清单（`PROVIDER_CONFIGS`）、请求构造、SSE 解析、工具调用循环全部放在 `commands/llm.rs` 一个文件里（现约 131KB），不拆分。配套建立 `docs/api-manuals/` 存放各家 API 的核对手册（`13da6f1` 起）。
- **为什么**：各家 API 的差异（认证、多模态格式、思考模式、缓存参数）是这个项目 bug 最密集的地带，集中在一处 = 改任何服务商行为只有一个去处，diff 一目了然；手册沉淀核对结论，防止同一个坑踩两次。
- **代价与边界**：文件巨大，新人（包括你）容易望而生畏——但它内部是按"配置表 → 构造 → 发送 → 解析"分节的，配合《02-关键数据流》的流 1 看并不可怕。

### A4. 前端路由用 hash 模式（立项之初）

- **决定**：`createWebHashHistory`，地址长如 `index.html#/settings`。
- **为什么**：桌面应用里页面是从本地文件加载的，没有 Web 服务器帮忙重写 URL，history 模式刷新会 404；hash 模式纯前端解析，天然兼容。

### A5. Naive UI 做底座，视觉上全面覆盖为黑白编辑设计系统（`94fdf35` → `ef07363` 推翻重来）

- **决定**：保留 Naive UI 的组件功能（表单、弹窗、表格），但视觉全部重写：无彩色、无圆角、指定字体与缓动，token 唯一权威源是 `src/styles/variables.scss`。
- **为什么**：早期用默认组件样式甚至统一过圆角（`94fdf35`），后来确立"黑白编辑"审美后整体推翻（`ef07363`）——差异化的视觉身份比跟随组件库默认值更重要。
- **代价与边界**：任何 UI 改动必须先读 `design-system` skill，否则很容易带进彩色/圆角破坏一致性；表单必须写中文 placeholder（`81eee83` 修过回落成英文的 bug）。

## B. 工程约定（多数是血泪史换来的）

### B1. 流式响应与大文件下载禁用总超时，只用读间隔超时（`840b1f0`）

- **决定**：凡是流式/下载场景，不设"整体必须在 N 秒内完成"的总超时，只设"两次数据之间不得超过 N 秒"的读间隔超时。超时常量集中在 `commands/constants.rs`（`d9382b7`）。
- **为什么**：长回答、大模型下载的总时长天然不可预测，任何固定总超时都会误伤正常慢请求——历史上一次修过**五处**这种误伤。
- **代价与边界**：排障口诀——故障卡在固定整数时长（如恰好 60 秒断），先搜自家 timeout 常量。

### B2. 不建常规测试套件，门禁 = 两次编译 + 一个冒烟测试（持续决策）

- **决定**：正确性门禁是 `pnpm build`（vue-tsc 类型检查）+ `cargo build`；唯一自动化测试是 Agent Team 冒烟测试（`workspace_smoke_test.rs`，环境变量触发）。此外靠 `self-test` skill 驱动真实 UI 做整体测试。
- **为什么**：单人 + AI 高速迭代阶段，全面测试套件的维护成本会吃掉迭代速度；类型系统（TS + Rust）本身就挡住了一大类错误。
- **代价与边界**：兜底偏薄，重构时缺回归保护。**已有计划**：从 workspace 模块开始补测试，兼作学习材料（2026-07-17 约定）。

### B3. 报错提示统一：左下角弹窗、全中文、说人话（`5776a22`、`4610c1b`）

- **决定**：所有提示/报错/警告走统一的左下角弹窗机制；后端错误经 `friendly_err` 包装成中文人话（如"保存会话失败，请重试"），原始错误进日志不进界面。
- **为什么**：散落的 alert 和英文报错堆是体验硬伤；用户需要的是"怎么办"，开发者需要的细节在日志里。

### B4. 单实例 + 关闭进托盘（`26c949b`、`f9f5023`）

- **决定**：重复启动只唤醒已有窗口；点关闭默认最小化到托盘而非退出（可设置），配全局快捷键唤起。
- **为什么**：一是 WebView2 profile 有锁，双开必炸；二是无人值守定位（定时任务、Agent 常驻）要求进程活着，用户"随手关窗口"不该杀掉后台循环。

### B5. 对外分发链：GitHub Releases 自动更新 + Beta 通道 + CI 三平台（`3b56603`、`ff6a569`、`a451eed`）

- **决定**：tag 触发 CI 自动出三平台安装包 + Windows 便携版 exe；应用内置更新检测，Beta 版走独立检测端点，版本比较用真实语义化比较（`c27d025` 修过"不等于就算新版"的 bug）。规划中：SignPath 免费签名 + Microsoft Store 消除 SmartScreen 警告。
- **为什么**：单人项目，发布流程必须零人工；Beta 通道让尝鲜用户帮忙暴露问题。
- **配套约定**：功能 PR 要顺手提版本号，别等打 tag 时才补（连漏过两次后立的规矩）。

## C. 产品方向（项目的"宪法"）

### C1. 定位转向：从"多服务商聊天客户端"到"无人值守自动化闭环"（2026-07-12 确立，README 定位重写见 `abf8bc4`）

- **决定**：项目主线是 **Agent Team 编排 → 定时任务触发 → 结果推送钉钉/飞书** 这条闭环；聊天、RAG、MCP 降级为入口和能力零件——不删不砍，但不再是主角。
- **为什么**：“客户端该有的功能都要有”是红海，无差异化；无人值守自动化闭环才是护城河。
- **代价与边界**：**新功能立项标准随之改变**——"是否让无人值守自动化更强/更可靠"，而不是"别家客户端有没有"。新增 LLM 服务商、向量索引优化等默认冻结，除非挡住主线。

### C2. Agent Team 的核心设计：常驻循环 + SQLite 消息总线 + 屏障式会议（`7eb4072` 初版 → `a32c888` 会议重构 → `0c70274`/`529b36b` 两轮结构性修复）

- **决定**：每个 Agent 是 Rust 里一个常驻 tokio 循环（平时睡、notify 唤醒）；Agent 间通信走 SQLite 表而非内存队列；会议采用屏障协议（到齐才开、轮流发言）；敏感操作（工具调用、创建 Agent、休眠）走 pending 事件等用户裁决。设计文档在 `docs/AgentTeamMode/`。
- **为什么**：写表通信可回放、重启不丢；屏障会议解决了初版抢话混乱；pending 裁决保证"自动化但人有最终否决权"。
- **代价与边界**：跨进程重启时，内存里的等待通道会死——所以启动时要重挂循环、把遗留 pending 标过期（见流 2 的重启恢复）。这个模块经历过 21 项清单级复审修复，是全项目打磨最狠的部分。

### C3. 定时任务生来就是 Agent Team 的扳机（`0a5c63b` → `78fccac` 独立成模块）

- **决定**：定时任务的动作本质只有一个——"到点把一条预设消息注入某个工作组"，而不是做成通用任务运行器。
- **为什么**：它是闭环的第一环，复杂度全部让 Agent Team 承接，调度器本身保持傻瓜（30 秒扫表，见流 3）。

### C4. 内置网页搜索/抓取工具，默认可用（`c819baf`、`bb34db7`）

- **决定**：不依赖用户配置 MCP，应用自带网页搜索和抓取两个内置工具，Agent 开箱即用；MCP 缺依赖时给安装提示而非静默失败。
- **为什么**：配 MCP（要装 Node/Python）对普通用户门槛太高，而"会上网"是 Agent 有用的底线能力。

### C5. 许可证：MPL-2.0 + 商用补充条款（`f61b71e` → `2ed5a45` 1.1 版）

- **决定**：个人/公司内部免费用不限；套壳分发、SaaS 托管需要商业授权，判定标准借鉴 Commons Clause。
- **为什么**：开源获取信任和社区，同时防"白嫖成竞品"。

### C6. 面向中文用户（贯穿始终）

- **决定**：UI、报错、注释、提交信息、文档全中文；服务商清单里国产厂商（智谱/千帆/豆包/Kimi/MiniMax/阿里…）与国际厂商并重；送达渠道选钉钉/飞书而非 Slack。
- **为什么**：目标用户画像明确是中文个人用户/小团队，B 站是主要宣传阵地。
