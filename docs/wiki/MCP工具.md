# MCP 工具

MCP（Model Context Protocol）模块让大模型调用外部工具：读写文件、操作浏览器、执行 Git 命令、生成 Office 文档等。

## 支持的连接协议

| 协议 | 适用场景 |
|---|---|
| **Stdio** | 本地进程方式运行的 MCP 服务器（最常见） |
| **HTTP** | 远程 / 本地 HTTP 服务 |
| **SSE** | 基于 Server-Sent Events 的服务 |

## 工作方式

1. 在 **MCP（模型工具）** 页添加并启动服务器
2. 应用启动服务器时**自动枚举其提供的工具**
3. 对话中启用后，工具以 LLM **原生函数调用**（Function Calling）方式透明代理给模型
4. 支持**多轮工具循环**：模型可以连续调用多个工具、根据结果再决定下一步，直到得出最终回答

## 预设一键安装

内置 10 个常用 MCP 服务器预设，一键安装无需手写配置：

- 文件系统（读写本地文件）
- 记忆（跨会话记忆）
- 网页抓取
- Playwright 浏览器自动化
- 时间
- Git
- Office 三件套（Word / PPT / Excel）
- ……

> 💡 Stdio 类预设通常依赖 Node.js（npx）或 Python（uvx）运行环境。缺少依赖时应用会弹出提示并说明安装方法。

## 内置网络工具

网络搜索与网页抓取已内置为应用自带工具，不依赖外部 MCP 服务器即可在对话中使用。

## 在哪里使用

- **对话**：Chat 输入区启用 MCP 服务器
- **Agent Team**：每个 Agent 可独立绑定 MCP 服务器，见 [[Agent Team协作团队]]
- **Skill**：技能可绑定 MCP 服务器，激活技能时一并生效，见 [[Skill技能]]
