# 定时任务（Scheduler）

Cron 定时任务是独立的侧边栏模块，全局管理所有定时任务，最典型的用法是**定时唤醒 Agent Team 里的 Agent 自动干活**。

## 四种调度类型

| 类型 | 说明 |
|---|---|
| **单次（once）** | 指定时间执行一次 |
| **间隔（interval）** | 每隔固定时长重复执行 |
| **每天（daily）** | 每天固定时刻执行 |
| **每周（weekly）** | 每周固定星期与时刻执行 |

## 与 Agent Team 集成

- 任务可绑定 **工作组 + 目标 Agent**，到期时自动向该 Agent 发送消息将其唤醒
- **广播模式**：目标 Agent 留空（`target_agent_id = null`）时，消息广播给工作组的**全部 Agent**
- 定时触发的活动在工作组时间线中以 `scheduled_trigger` 专属条目展示

## 运行机制

- Rust 后端**每 30 秒轮询一次**到期任务
- App 运行期间全自动触发，无需人工干预
- ⚠️ 定时任务依赖 App 处于运行状态——App 关闭期间错过的任务不会补触发

## 快速入口

在 **Agents（协作团队）** 页点击**时钟图标**，可直接跳转到定时任务页并自动预填当前工作组的筛选条件。

## 典型场景

- 每天早上 9 点唤醒「资讯 Agent」抓取并汇总行业新闻
- 每小时触发「监控 Agent」检查某个服务状态
- 单次定时：今晚 10 点让 Agent 提交一份日报
