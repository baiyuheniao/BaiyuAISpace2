# 本地部署（Local Deploy）

Local 模块管理本地模型运行时，完全离线可用、数据不出设备。分 **Ollama / LM Studio / Docker** 三个 Tab。

## Ollama（全托管）

从安装到跑模型全程在应用内完成，不用碰命令行：

- **安装管理**：自动检测是否已安装；未安装可一键下载安装（内置多镜像源，国内网络友好）
- **服务管理**：启动 / 停止 Ollama 服务，查看版本
- **模型管理**：
  - 在线搜索模型（含各尺寸 tag，如 `qwen3:8b` / `qwen3:32b`）
  - 下载模型（大文件下载不设总超时，只按读间隔判断中断，弱网也能拉完大模型）
  - 删除本地模型

装好模型后，到 **Settings（设置）** 添加 Ollama 类型的 LLM API 配置即可在对话中使用。

## LM Studio

适合喜欢 GUI 管理 GGUF 模型的用户（需自行安装 [LM Studio](https://lmstudio.ai/)）：

- 连接状态检测（走 LM Studio 的 OpenAI 兼容接口）
- 模型下载
- 模型加载 / 卸载

## Docker 部署

面向想用容器跑推理服务的用户（需自行安装 Docker）：

- **镜像管理**：拉取、删除
- **容器管理**：启动、停止、删除
- **内置部署预设**：Ollama、LocalAI（CPU 版 / GPU 版）三个一键部署模板

## 该选哪个？

| 方案 | 适合人群 |
|---|---|
| **Ollama** | 大多数用户的首选——应用内全托管，零门槛 |
| **LM Studio** | 已在用 LM Studio、想精细控制 GGUF 模型加载的用户 |
| **Docker** | 熟悉容器、想部署 LocalAI 等自定义推理服务的用户 |
