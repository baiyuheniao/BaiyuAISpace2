# 知识库（RAG）

RAG 模块把你的文档变成可检索的向量知识库，对话或 Agent 回答时自动引用相关内容。

## 前置条件

需要先在 **Settings（设置）** 页配置 **Embedding API**（用于文档向量化）。如需精排还可以配置 **Reranker API**（Cohere 兼容接口）。

## 文档导入

支持的格式：

- **办公文档**：PDF、Word（.docx）、PowerPoint（.pptx）、Excel（.xlsx / .xls / .csv）
- **文本**：Markdown、HTML、TXT
- **代码**：常见编程语言源文件

导入后文档自动解析、分块、向量化入库。

## 检索机制

- **混合检索**：向量相似度 + 关键词匹配双路召回
- **Reranker 精排**（可选）：配置 Cohere 兼容 Rerank API 后，对召回结果二次排序，显著提升相关性
- **可调参数**：
  - 分块大小 / 分块重叠（导入时生效）
  - Top-K 返回结果数

## 在对话和 Agent 中使用

- **对话**：Chat 输入区勾选要挂接的知识库，发送消息时自动检索并注入上下文
- **Agent Team**：每个 Agent 可独立绑定知识库，见 [[Agent Team协作团队]]
- **Skill**：技能可绑定知识库，激活技能时一并生效，见 [[Skill技能]]

## 规模限制（Beta 已知事项）

当前向量检索为**全量扫描**，暂不适合超大规模数据：

- 单个知识库建议控制在 **约 1 万个文本分块以内**（约几百份常规文档）
- 超出后搜索会明显变慢
- 向量索引优化已在路线图中（见 [Issue #18](https://github.com/baiyuheniao/BaiyuAISpace2/issues/18)）

实践建议：按主题拆分成多个知识库，对话时只挂接相关的那个，既快又准。
