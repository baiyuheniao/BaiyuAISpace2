## 第三条 定义 / Article 3 Definitions

**判断的核心标准 / The Core Test**：不看修改了多少代码，而看**商业价值的实质来源**——你对外销售或提供的产品/服务，其主要功能与价值是来源于你自己的开发成果，还是完全或实质上来源于本软件本身。（本标准参考 Commons Clause 中 "value derives, entirely or substantially, from the functionality of the Software" 的判断方法。）

The test is not how much code you changed, but **where the commercial value substantially comes from**: whether the product or service you sell derives its primary functionality and value from your own development, or entirely/substantially from this software itself. (This test follows the approach of the Commons Clause: "value derives, entirely or substantially, from the functionality of the Software.")

**"实质性二次开发" / "Substantial Derivative Development"**：在本软件基础上形成了独立的技术价值与产品形态。示例（满足其一即可能构成，非穷尽）：新增了本软件不具备的核心能力并以其为主要卖点；面向明显不同的用户场景进行重构；将本软件作为组件集成进一个功能更大的自有系统。

Building independent technical value and product identity on top of this software. Illustrative (non-exhaustive) examples: adding core capabilities this software does not have and marketing them as the primary selling point; restructuring for a clearly different user scenario; integrating this software as one component of a larger system of your own.

**"套壳/换皮" / "Reskinning/Wrapping"**：保留本软件的核心功能与架构，仅修改名称、界面、品牌标识、默认配置，或加以简单封装（如加登录、加收费墙、加安装器），即作为商业产品/服务对外销售——即其价值完全或实质上仍来源于本软件本身。

**"托管服务" / "Hosted or Managed Service"**：以在线服务形式向第三方提供对本软件全部或实质性功能集的访问（参考 Elastic License 2.0 对 managed service 的界定），无论是否对本软件进行过修改。
